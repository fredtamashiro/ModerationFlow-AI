# Implement Spec Command

Use this command when implementing a spec.

## Instructions

Read these files first:

1. `AGENTS.md`
2. `docs/development-runbook.md`
3. `.agents/docs/project-context.md`
4. `.agents/docs/code-standards.md`
5. `.agents/docs/implementation-recipe.md`
6. The active spec in `.agents/specs/`

## Implementation Rules

- Implement only the active spec.
- Do not add unrelated features.
- Do not refactor unrelated code.
- Preserve working authentication and infrastructure.
- Keep changes small and reviewable.
- Validate backend and frontend after changes.
- If something is ambiguous, document the assumption.

## Required Final Summary

At the end, provide:

- Summary of changes;
- Files changed;
- Validation commands run;
- Known issues or limitations;
- Recommended next spec.
